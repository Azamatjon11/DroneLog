DroneLog

A real-world, multi-platform UAV flight telemetry dataset for airframe identification and payload state recognition — with benchmarks for classification, regression, navigation, and event detection.

📦 Dataset (Zenodo) 💻 Devkit (GitHub) 📄 Paper

Overview

DroneLog is collected from five real quadrotor platforms spanning five frame sizes (250 mm–680 mm) and two payload-capable platforms (Holybro X500 V2 and X650) flown outdoors under naturalistic, unscripted conditions. All data is released as native PX4 ULog flight logs, requiring no external cameras, motion capture, or additional sensing hardware.

  • 5 quadrotor platforms across 5 frame sizes (250 / 450 / 500 / 650 / 680 mm)
  • 5 payload behavioral scenarios across multiple weight configurations (0.5–2 kg)
  • Full PX4 telemetry per flight: IMU, GPS, attitude, local position, battery, actuator outputs
  • Ground-truth event timestamps for payload pickup and drop
  • Released in both native ULog and pre-extracted CSV formats

Benchmark Tasks

The devkit ships five runnable baseline tasks covering four distinct problem types:

1 · Frame-Size Classification

Identify airframe size class from windowed IMU statistics.

2 · Payload-Scenario Classification

Classify behavioral scenario (no payload, pickup, drop, etc.).

3 · Payload-Mass Regression

Estimate carried payload mass as a continuous value.

4 · Navigation / Odometry

IMU-only strapdown integration scored against GPS and EKF tracks.

5 · Pickup/Drop Event Detection

Detect the exact moment of payload state transitions.

Getting Started

Install the devkit and run any baseline directly from the command line:

source venv/bin/activate
pip install -r requirements.txt

python scripts/task1_frame_classification.py --data . --out outputs
python scripts/task4_navigation_odometry.py --data . --out outputs

Every script supports --inspect to preview topics and matched fields before running a full benchmark. See the repository README for full usage details.
